Metric Tile

One component for every metric card on the platform. Header and value are identical everywhere; only the container (grouped / standalone / inline) and the footer (caption / status dot / benchmark / delta) change per use case. The value is always neutral — status is communicated only through an indicator, never by recoloring the number.

Inline — page header KPIs
Revenue (YTD) 94 thousand kr. n/a

Disposable liquid. 17,149 thousand kr. +52.5%

No card, no border — sits directly on the page background, for KPIs living in a page header/toolbar alongside other prose (breadcrumb, CVR, last-updated). This is the one place the label stays sentence-case and sits inline before the value, instead of caps-above-value like every other variant. The trailing tag is neutral n/a when there's nothing to compare against — never a placeholder word — and only turns green/red (--komma-status-success/critical) for an actual delta. The value uses --komma-font-sans, not the display font: per the type scale, Inter Tight is reserved for large KPI figures (--komma-metric-*), and this value is deliberately smaller than those.

The status dot uses the generic --komma-status-* tokens (.mt-dot--ok/--warn/--crit/--info). Callers map their own domain state (e.g. liquidity health) onto one of these four — the component itself stays domain-agnostic. The value text stays neutral even for a qualitative answer like "No". Icons are Lucide (info / pencil), inlined as raw SVG so the tile has no JS dependency — copy the <svg> markup as-is when reusing this pattern.
